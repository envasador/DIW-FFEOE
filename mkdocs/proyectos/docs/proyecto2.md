# **Proyecto 2: Diseñar para que se entienda**

### **¿De qué va esto?**

Ya tienes la estructura funcional. Sabes qué hace tu producto, cómo funciona, qué pantallas tiene y cómo se conectan. Ahora toca que se **entienda visualmente**.

Este proyecto transforma tus wireframes grises en un diseño visual coherente, profesional y escalable. Aquí defines la personalidad visual de tu proyecto: qué colores usas y por qué, qué tipografías comunican mejor tu mensaje, cómo organizas el espacio para crear jerarquía, qué iconos ayudan al usuario sin confundirlo, y qué componentes se repetirán consistentemente en toda la interfaz.

No se trata de hacer las cosas "bonitas", sino de tomar decisiones visuales basadas en principios, no en gustos personales. Construirás un sistema de color accesible y funcional, elegirás tipografías que realmente se lean bien en pantalla, documentarás todo en una guía de estilo que sirva como fuente única de verdad, y crearás una biblioteca de componentes reutilizables que garantice coherencia en toda la interfaz.

Al final de este proyecto tendrás mockups de alta fidelidad listos para testear y para que desarrollo los implemente. Y lo harás creando un sistema, no diseñando pantallas sueltas: cada decisión se documenta, se reutiliza, y se escala conforme el proyecto crece.

### **Resultados de Aprendizaje y Criterios de Evaluación**

Este proyecto trabaja principalmente sobre el **RA1** y el **RA6**:

**RA1: Planifica la creación de una interfaz web valorando y aplicando especificaciones de diseño.**

Los criterios de evaluación que aborda este proyecto son:

* Se ha valorado la importancia de definir y aplicar la guía de estilo en el desarrollo de una aplicación web 
* Se han utilizado y valorado distintas tecnologías para el diseño de documentos web 
* Se han creado y utilizado plantillas de diseño

**RA6: Desarrolla interfaces Web amigables analizando y aplicando las pautas de usabilidad establecidas.**

Los criterios específicos que trabajamos son:
* 
* Se ha modificado el interfaz web para adecuarlo al objetivo que persigue y a los usuarios a los que va dirigido 
* Se han verificado los criterios de accesibilidad y diseño aplicados
